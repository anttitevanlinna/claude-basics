# Claude Basics

An agentic training for IT teams rolling out Claude. Participants learn Claude BY using Claude.

**3 x 45-minute modules.** Pre-work and post-work handled by Claude. The facilitator is only needed for the part where humans need to be in a room together.

## The Arc

**Make something -> Verify it -> Look ahead**

| Phase | What happens |
|-------|-------------|
| Pre-work | Get set up. Build your first personal system. Write your CLAUDE.md. |
| Module 1: "Stop chatting -- build a system" | Build a working system with guardrails for a real task. Personal exercise. |
| Module 2: "Can you be trusted to run this?" | Verify output. Detect hallucinations. Harden the system. |
| Module 3: "Where is this all going?" | Find the crux of your rollout. Multi-agent group exercise. Buyer synthesizes. |
| Post-work | Buyer measures adoption. Claude keeps the learning alive. |

## What makes this different

- **Agentic training.** Claude is a co-participant, not just the subject. Every exercise runs in Cowork.
- **Continuous artifact.** One system, born in Module 1, stress-tested in Module 2, evolved in Module 3. Participants walk out with something they'll use Monday.
- **Buyer as active participant.** The IT Director composes instructions, tests ground truth, ranks challenges, and synthesizes the final rollout strategy. Not a spectator.
- **Multi-agent exercise.** Module 3's group exercise is a multi-agent system: individual agents, shared workspace, synthesizer. They build it without being taught it.
- **No slides.** No slides-only sections. Every concept demonstrated live or practised immediately.

## Two fundamental questions

- **C: "Can you be the person your org trusts to run this?"** -- the spine
- **D: "Where is this all going?"** -- the accelerant
